There is one more thing we need to fix on the AWS server, and this will 100% solve it!

When we updated all the IPs in the AWS backend to `54.206.184.54`, the AWS server started trying to connect to its **own public IP** to read the video frames from Redis. AWS servers block themselves from doing this (it's called Hairpin NAT). The server MUST use `127.0.0.1` (localhost) to talk to its own database.

Run this **exact block of code** in your AWS terminal. It will rebuild the backend `.env` file correctly, allowing the server to read the frames, and then restart the stream!

```bash
cd ~/SmartGuard/smartguard_backend

cat << 'EOF' > .env
DATABASE_HOST=127.0.0.1
DATABASE_NAME=smartguard
DATABASE_USER=sg_admin
DATABASE_PASSWORD=Techroque123!
DATABASE_PORT=5432

REDIS_HOST=127.0.0.1
REDIS_PASSWORD=Techroque123!
CLOUD_REDIS_HOST=127.0.0.1
STREAM_BASE_URL=http://54.206.184.54:8001
EOF

sudo systemctl restart daphne
sudo systemctl restart sg-video
```

Once you run this, your local laptop will continue pushing frames perfectly to `54.206.184.54`, and the AWS server will read them perfectly from `127.0.0.1` (itself), and your website will stream it!
